# /lcw file <name>

把查询中结晶出的洞察沉淀为 wiki 内容。

**关键区分**：file 归档的内容有两种来源，处理方式不同：

## 来源一：代码事实的结晶

查询或对话过程中，从代码中提炼出了值得独立记录的高阶理解（如跨模块的隐藏耦合、重复模式、未文档化的业务规则）。这些信息**来源于代码**，只是需要人提问才被结晶出来。

**处理方式**：融入已有的 wiki 页面——
- 跨模块关系 → 补充到相关 `modules/{repo}/` 和 `domains/` 页面
- 业务规则 → 补充到 `domains/` 页面
- 接口发现 → 补充到 `interfaces/` 页面
- 新发现的问题 → 创建 `issues/` 页面
- 新发现的业务流程 → 创建 `flows/` 页面

如果内容确实不适合融入任何已有页面，说明可能发现了新的领域或模块——应该创建对应的 `domains/` 或 `modules/` 页面。

## 来源二：人类提供的非代码信息

用户在对话中提供了代码中不存在的信息——业务背景、历史决策原因、团队约定、口头知识等。

**处理方式**：存入 `notes/`，按 SCHEMA.md 的 notes 模板创建页面，标记为 `status: unverified`。不直接写入代码事实页面。

## 写入后

1. 添加 `[[wikilink]]` 到相关页面
2. 在被引用页面补充反向链接
3. 检查词汇表，补充新术语
4. 更新 `index.md`
5. 写 `log.md`
